#114 🟢 LOW Kernel Security › Weak Kernel Hardening /proc/sys/kernel/dmesg_restrict ▼
Path / Target
/proc/sys/kernel/dmesg_restrict
📋 Description
dmesg restriction is set to 0 (recommended: 1). When set to 0, unprivileged users can read kernel messages which may leak addresses.
🔗 Reference
Linux kernel hardening documentation
🛠 Mitigation
Restrict dmesg: echo 1 > /proc/sys/kernel/dmesg_restrict
  Persist: add 'kernel.dmesg_restrict=1' to /etc/sysctl.conf
